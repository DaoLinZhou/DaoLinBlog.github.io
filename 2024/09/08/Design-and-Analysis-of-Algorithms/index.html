Design and Analysis of Algorithms

Stable employer-application assignment

有一些雇主和一些申请人, 我们要把申请人 assign 给雇主使得对于每个 雇主 $E$ 和 每个没有和 E 配对的申请人 $A$ , 下面两个条件中至少有一个 满足

  1. $E$ 更倾向与它发送其他 offer 的申请人
  2. $A$ 更倾向与它当前的雇主而不是 $E$

即不会出现 A 和 E 都倾向与对方, 而我们却没有将二者配对的情况

这种情况我们称为 stable matching

将问题简化一下:

每个申请人会向所有的雇主发出申请, 而每个雇主只会接受一个申请人


设计算法

G-S Algorithm for Stable Matching

  1. 首先初始状态所有的 element 都是 unpair 的

  2. 对于每个 unpaired $m$, 让 $W_m = \{ w|\text{W and m has not tried to pair} \}$ 并让 $w \in W_m$ with largest $f_m(w)$

    如果 w 是 unpaired 那么将 $(m,w)$ 放入 pair 中

    不然则 unpair $(m’, w)$

    ​ 如果 $g_w(m)>g_w(m’)$ 那就 remove pair $(m’, w)$ 并让如 $(m,w)$ pair

    ​ 不然则 mark $w$ as that $m$ has tried to pair

  3. 最后当所有元素都是 paired 时, 算法停止

每个 m 都有一个 W 的 copy $W_m$, 用于记录自己是否尝试和 w 匹配过

每次 m 从 $W_m$ 中取出一个 $w$, 尝试配对, 如果 w 也没有配对则将他们配对

如果 w 有配对则看看当前配对是否更合适


Theorem: G-S Algorithm 在终止时会给出 stable matching

证明: 由于 $S$ 是一个 matching 并且 $|M|=|W|=n$. 根据 Property 1: (当 $w$ 被 pair 以后, 在这个算法的执行过程中, 他会保持 paired)

如果有一个 unpaired $m$, 那么就有一个 unpaired $w$, 并且 m 没有 尝试去与其匹配过. 所以 每个 $m$ 都会和一个 $w$ 配对, 并且 这个算法终止时会返回一个 perfect matching $S$

反证法, 假设有 instability,


  • 这个算法的 iteration 可能会接近 $n^2$
  • 对于一个 input 可能有多个 stable matching
  • G-S algorithm 可能会已不同的 order 选取 $m$ 来得到 different pairs (in different execution)

Notations:

算法有很多notation, 大致意思就是

Greedy


Dynamic Programming

对比两个 solutions, 当 一个 solution 的 weight 更低但可以获得更高的 value 那就说明他 dominates 另一个 solution

当 set 中出现一个 solution $A$ dominate 另一个 solution $B$, 我们就将 $A$ 删除

而当两个 solution 互相 dominate 时 (有同样的 value 和 weight) 那么随便删除一个就好
