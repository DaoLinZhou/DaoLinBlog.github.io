如何子字符串中查找子串

最简单的方法是暴力匹配

很显然, 这个算法的复杂度是 $O(MN)$ 级别的

  • $M$ 是 str1 的长度 (for loop)
  • $N$ 是 str2 的长度 (if statement 对比两个字符串是否相等)

可以看到, 每次对比失败后, 主串中的索引会倒退到下一个位置


KMP 的核心思想就是让主串中的索引不进行倒退, 它只往前走.

放在while loop 的末尾是没有问题的, 我们先判断当前字符是否一致, 之后判断是否匹配成功. 就是说, 只要有字符进行匹配我们就会判断一下是否匹配成功. 不会漏判


是否可以吧这两行代码放在 while loop 的开头? 不可以

因为如果在一次匹配中, i 和 j 同时到达末尾, 此时匹配成功, 然而 i 在自增后不会进入 while loop, 我们错过了判断匹配成功的时机. 例如匹配两个一样的字符串

如果我们对 while loop 的condition 稍作修改, 改为 while i <= len(string): 那么此时应该是可以把这两行代码放到 loop 的开头, 然而整体代码的语义又很难解释



Next 数组生成

next 数值代表在匹配失败的时候, 子串中可以跳过匹配的字符个数

为什么可以这么做?

下面例子中的前4个字符, 拥有共同的前缀和后缀 AB (最长的前后缀)

也就是说, 当我们跳过这两个字符后, 我们默认比较了原本的前缀和后缀(因为他们是一样的). 这就节省了重复运输的时间


使用最长的共同前后缀来跳过更多的字符.


前后缀不能是字符串本身, 因为这样主串和子串的 index 都没有改变, 依旧不匹配, 形成无线 loop. 没有意义

但是前缀和后缀可以有重叠, 例如 :

A A A A
0 1 2 3

AAAA 有相同的前后缀 AAA
