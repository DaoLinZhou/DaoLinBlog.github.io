线段树

线段树 Segement tree 也被称为区间树 Interval tree


为什么要使用线段树?

线段树到底解决了一个什么样的问题?

对于有一类问题, 我们关系的是线段(或者区间)


经典的线段树问题: 区间染色

有一面墙, 长度为n, 每次选择一段墙进行染色

问m次操作后, 我们可以看见多少种颜色? m次操作后, 我们可以在[i, j]区间内看见多少种颜色?


另一个经典问题: 区间查询

我们有时候会对一个区间内的元素进行统计查询

例如: 查询一个区间[i, j]的最大值, 最小值, 或者区间数字和


这里有两个操作:

  1. 染色操作 (更新区间)
  2. 查询操作 (查询区间)

显然我们明显可以使用数组来实现. 而数字在这两个操作的时间复杂度都是O(n)级别的

然而当我们动态使用O(n)级别的操作时. 它的时间复杂度是较高的.

而线段树则可以在O(logn)的时间内完成

使用数组 使用线段树
更新 O(n) O(logn)
查询 O(n) O(logn)


对于线段树我们是不考虑添加元素或删除元素.

对于大多数线段树的问题, 区间本身是固定的



线段树的基础表示

8正好是2的3次方, 所以生成的线段树是一个满二叉树

然而如果线段的长度为10, 此时生成的二叉树就不是满二叉树了.

当不能数组平分时, 我们就必须要让一边少一些, 另一边多一些

而且线段树也不一定是一个完全二叉树(因为最后一排的节点没有靠左)


不过线段树是平衡二叉树(树的最大深度和最小深度只能为1)


而平衡二叉树也可以用数组表示. 我们可以把它看作是一个满二叉树

只不过最后一层很多节点都不存在, 这样就可以用数组表示.


如果区间有n个元素, 那么用数组表示需要有多少节点?

对于满二叉树, 最后一层的节点数等于前面所有节点数之和+1.

如果n比较好, n = 2k, 则只需要2n-1的空间(2n也可以) $2*2^k-1$


而如果n不等于2k 而是n=2k + m, 那么就需要再加上一层. 则需要 $42^k-1=22^{k+1}-1$ 的空间, 或者用4n也是可以的.



线段树还可以设计为懒更新.

就是更新时不立即更新, 而是等下一次操作(查询或更新时)再进行更新


二维线段树

上面的线段树本质是一维线段树

什么是一维线段树?

就是处理的区间是在一维空间中(一个坐标轴中).


我们可以把思想扩展到二维空间中.

把二维空间分成一个个小块.

同理3维乃至高维都可以这么做


线段树本身是一种思想: 使用树这种数据结构将大的单元拆分成小的数据单元. 递归地表征这些数据
