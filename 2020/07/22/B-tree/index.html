B树

B-tree, B-树

B-树

B树是一种平衡的多路搜索树

3阶B树 (一个节点最多有3个子节点)

4阶B树 (一个节点最多有4个子节点)

5阶B树 (一个节点最多有5个子节点)



B树的性质

对于一个m阶B树, 假设一个根节点存储的元素个数为x

  • 根节点的元素个数为 $1 \le x \le m-1$
  • 非根节点的元素个数 $ceil(m/2)-1\le x \le m-1$
  • 如果有子节点, 子节点的个数为 y = x + 1
    • 根节点: $2\le y \le m$
    • 非根节点: $ceil(m/2)\le y\le m$

当m=3时, $2\le y \le 3$, 可以称为(2, 3)树, 2-3树

当m=4时, $2\le y \le 4$, 可以称为(2,4)树, 2-3-4树

当m=5时, $3\le y \le 5$, 可以称为(3,5)树, 3-4-5树

数据库使用B类树, B-tree或B+tree

一般使用200~300阶的B树



添加元素

新添加的元素必定是添加到叶子节点. (和最后一层的某个节点融合)

因为在二叉搜索树添加元素时, 我们就是添加到叶子节点, 而B树在某种程度上和二叉搜索树等价


然而和节点融合后, 可能节点的元素会超过限制. 这种现象称为上溢


上溢的解决(假设5阶)

5阶B树中, 每个节点最多存储4个元素, 当存储5个元素时就会上溢

上溢节点的元素个数必然等于 m (m阶B树)

先求出上溢节点最中间元素的位置. 假设为k. 例如上图中k=2.

之后将k位置的元素向上与父节点合并. 再将**[0, k-1] 和 [k+1, m-1]**位置的元素分裂成两个子节点


此时父节点可能再次上溢

如果父节点上溢, 则重复这个过程. 例如上图中就要把40和它的父亲节点融合, 再进行分裂

极端的情况可能会一直分裂到根节点.

只有上溢到根节点, B树才会长高


然而当m为偶数时, m阶B树上溢时如何选取中间元素? 中间位置有两个元素, 这两个元素选哪个作为中间元素都可以. 让它和父节点融合.



删除元素

对于删除叶子节点, 直接删除即可

对于非叶子节点, 找到前驱或后继再进行删除, 转化为删除叶子节点


5阶B树有一个陷阱


这就是下溢(underflow)

下溢节点的元素数量必然等于 $ceil(m/2) - 2$


如果下溢节点的临近兄弟节点至少有 $ceil(m/2)$个元素, 可以向其借一个元素.

将父节点的元素b插入下溢节点的0位置

而再把 a 放到 b 原本的位置

这个操作实质就是旋转, 对b进行右旋操作

注意a的右孩子会变成b的左孩子


然而如果临近的兄弟节点无法借元素怎么办? ( 只有$ceil(m/2)-1$个元素)

此时应该将父节点的元素 b 挪下来和左右子节点进行合并

此时合并后的节点元素个数等于 $2*ceil(m/2)-2 \le m-1$

此时父节点又少了一个元素, 可以又会下溢, 依然要向上传播.

只有下溢到根节点, B树的层数才会减少



红黑树就是4阶B树(2-3-4树).

  • 所有节点能存储的元素个数x: $1\le x\le 3$
  • 所有非叶子节点的子节点个数: $2\le y \le 4$
