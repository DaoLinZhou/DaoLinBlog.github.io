图论(六)-最短路径

带权图还有一种常见问题: 最短路径问题

对于最小生成树问题通常都是作用于无向图

而最短路径问题则是同时支持无向图和有向图

所谓最短路径, 就是求出一个节点到另外一个节点耗费最小的路径

可以看出, 最短路径问题有很多实用方面

路径规划是最典型的一个例子

  1. 不仅是生活中走路的路径, 网络上路由于路由通信, 如何让文件从一点到另外一点

  2. 工作任务的规划, 本质也是路径规划, 每一个节点代表一项工作任务, 而边代表完成这个工作任务相应的耗费

  3. 游戏中自动寻路

而无权图中, 使用广度优先遍历时我们也说找到的路径是最短路径, 其实我们所找到的是从起始节点到其他所有节点的最短路径, 我们也将其称之为最短路径树(Shortest Path Tree), 其实这个树也是这张图的一个生成树, 所有的顶点距离起始的顶点的响应的权值是最小的, 而求得最短路径树的过程又被称之为解决**单元最短路径(Single Source Shortest Path)**问题, 单元(single source)就是起始点


松弛操作 (Relaxation)

对于有权图来说, 从一个点直接到达另一个点可能不是最短路径, 通过其他节点可能会有更短的路径. 而这个就是松弛操作

松弛操作是最短路径求解的核心


Dijkstra 算法

算法局限: 不能处理带有 负权边 的图

复杂度: O( E log(V) )

Dijkstra算法的思想

一个点的相邻边中最短的那条边就是这个点到达那个点的最短路径

例如下图, 节点0的最短相邻边是0->2, 权值为2. 所以0->2的最短路径就是0->2. 因为如果从0到其他节点(大于0->2)再走向2的话一定比直接走向2权值更大. 前提就是图中没有负权边

之后再寻找起始点到没访问过的节点中最小的权值对应的点, 2->1, 可以看到 0->2 + 2->1 的权值小于 0->1, 所以目前到1的最短路径就是0->2->1, 同理 如果2去往其他节点在回到1一定比 2->1 大. 重复这个过程.

可以从上面的逻辑中看到, 我们需要找到相邻边中的最短边, 所以要用到优先队列


dijkstra 的思想，就是 bfs！因为没有权值，bfs 用一个普通队列就够了；但有了权，为了处理权，dijkstra 就需要一个优先队列了。在此基础上，bfs 和 dijksta 近乎是一套逻辑

Bellman-Ford算法和负权边

Bellman-Ford可以处理具有负权边的图, 但这个图中不能有负权环.

如果一个图拥有负权环, 那么它就没有最短路径

Bellman-Ford 算法的思想

如果一个图没有负权环,

那么从一点到另外一点的最短路径, 最多经过所有的V个顶点, 有V-1条边

否则, 存在顶点经过两次, 即存在负权环


对一个点的一次松弛操作, 就是找到经过这个点的另外一条路径, 多一条边, 权值更小.

如果一个图没有负权环, 从一点到另外一点的最短路径, 最多经过所有V个顶点, 有V-1条边

对所有节点进行V-1次松弛操作, 就可以找到这一点到所有点的最短路径

如果还可以继续松弛, 说明原图中有负权环

时间复杂度 : O(VE)

这个算法不需要优先队列


好吧上面写的其实是错的.

因为不能保证进行一次松弛操作就会多一条边

真正的原因是这样: 首先有一个定理:

它说的是, 假设有一个最短路径, 我们根据最短路径上边出现的顺序进行松弛操作, 我们可以得到最短路径.

而一个最短路径最多有$V-1$ 条边, 所以我们直接对所有边进行 $V-1$ 次松弛操作, 这样就可以抱枕松弛操作的顺序是最短路径上边出现的顺序.

如果只关注从s到t之间的最短路径, Bellman-Ford 算法 无法提前终止

Bellman-Ford 算法可以优化 为 SPFA(Shortest Path Fast Algorithm).

使用一个队列, 只对关心的点的邻边进行松弛操作.

最差的时间复杂度依然是 O(VE)



Floyed 算法

Dijkstra 和 Bellman-Ford 都是求单元最短路径算法

而 Floyed 算法是求所有点对最短路径算法.


为什么要求所有点对最短路径?

这里举一个简单的应用, 也是图论领域的一个概念: 图的直径

图的直径是指: 在这个图中, 所有点对最短路径中的最大值


Dijkstra是求单元最短路径的算法, 如果把它重复v次的话, 就能得到所有点对的最短路径. 时间复杂度O(VElogE), 但是图不能包含负权边

同理Bellman-Ford 就是 $O(V^2E)$ 级别的, 可以处理负权边, 检测负权环

Floyed 是一个 $O(V^3)$ 级别的算法.

它比 Bellman-Ford 的复杂度要地, 如果是稀疏图的话会比 Dijkstra 算法复杂度高一些. 但是Dijkstra有不能处理负权边的限制, 而Floyed可以处理负权边, 检测负权环



Floyed 算法的思想

虽然Floyed算法本身没有用到松弛操作. 但是原理类似


松弛操作:

用虚线代表s-b, s-a, s-c, 当前已经找到的最短距离

在这个基础上 a-b 之间是一条边, 所以a-b之间用的是实线.


Floyed的思想类似, 区别只是, a-b 之间不一定只是一条边, a-c 之间也不一定只是一条边.

反正Floyed解决的是所有点对之间的最短路径问题. 所以也可以想成当前已经找到了从b到a的一条最短路径. 这条路径可能经过了多个顶点.

此时这个不等式的关系依然是存在的. 区别只是之前 b-a 代表的是一条边, 现在代表的是一条路径

此时floyed算法的基本原理就是这个样子. $dis[i][j]$ 代表i到j的最短距离


初始如果 v-w 有边, 那么$dis[v][w]=vw$, $dis[v][v]=0$, 否则 $dis[v][w]=\infin$

首先进行一重循环

在这重循环下再进行两重循环. 这两重循环是找最短路径的起始点和终止点

t是我们在找这条最短路径时, 从v到w是否可以多绕一个弯, 从v-t, t-w这样的距离是否更短.


为什么这样就能得到所有点对之间的最短路径?

最外层的循环, 每轮是在求解出中间经过 [0 … t] 这些点的最短路径

当t=0时, 整个3重循环就是说允许经过0时所有点的最短路径

当t=1时, 相当于又考虑一个点, 如果任意两个点之间还能让它经过1这个点的话, 对应的最短路径是多少. (可能经过0, 可能经过1, 也可能两个点都经过, 或者两个点都没经过)


如何检测负权环?

在经过3重循环之后, 就计算出了所有点对的最短距离.

此时Floyed也计算出来了自己到自己的最短距离

而对于一个没有负权环的图来说, 自己到自己的最短距离一定是0

然而如果有负权环的话, 自己到自己的距离是小于0的. $dis[v][v] < 0$

所以只要看一下二维数组dis的主对角线上的值, 如果有小于0的, 就说明有负权环

Dijkstra, Floyed 和 Bellman-Ford 都是有向图算法, 不过无向图可以看作特殊的有向图

不过处理无向图 Floyed 和 Bellman-Ford 就没有意义了, 因为他们的优点就是处理负权边, 然而无向图如果有负权边就一定有负权环


这三个算法都使用了动态规划

因为这三个算法都定义了一个状态, dijkstra还使用了贪心的思想
