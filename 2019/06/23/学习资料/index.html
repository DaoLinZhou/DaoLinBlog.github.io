源码

c++源码

首推 STL。我认为如果你只读一份 C++ 源码的话，近乎一定是 STL。只要写 C++，肯定在不断地用，对接口的使用熟悉，学习起来会更清晰；同时市面上的资料也很多，至少我那个年代是。

链接： http://www.rrsd.com/software_development/stl/stl…

在学习了 STL 源码之后，剩下的源码学习，如果需要的话，在我看来大多和领域相关。随便举一些例子。以下排名不分先后。

  1. boost。不用提了，很多 boost 实现后来都被纳入了 C++ 的标准库中。但我认为可读性不高，资料相对更少。链接： https://github.com/boostorg

  2. facebook 内部使用的 C++ 扩展库 folly。链接： https://github.com/facebook/folly

  3. OpenCV 这类 Open 的库。链接： https://github.com/opencv/opencv
    类似的还有 OpenGL，OpenAL 什么的。

  4. Chromium。学一学看一看就能换皮做红星浏览器了。。。（玩笑）链接：https://github.com/chromium/chromium

  5. redis。链接： https://github.com/antirez/redis

  6. qt： https://www.qt.io/download-open-source

上面的都是相对比较重的“大项目”，但可能你希望看的是一些轻量级的小项目。其实 github 上搜索 cpluscplus 相关的项目，有很多。

随便举个例子：这个项目是一个 csv 文件的解析器。这类项目也挺适合源码学习的。就一个文件。链接： https://github.com/ben-strasser/fast-cpp-csv-par…

最后祭出大杀器。github 上有一个项目，叫 Awesome CPP，总结了一票好的 C/C++ 开源项目。链接： https://github.com/fffaraz/awesome-cpp
