HTTP 基础

http是目前应用最广泛的网络协议之一, 许多服务器同时通过http来收发请求.

这篇文章将讲述一些基础的http知识

简介

HTTP是应用层的协议, 基于请求与响应模式无状态的应用层协议

HTTP1.1中给出一种持续连接机制 keep alive

超文本传输协议HTTP主要特点:

  • 支持客户/服务器模式 (客户端通过url向服务端发送web请求, 服务端根据接收到的请求返回响应信息)

  • 简单快速

  • 灵活

  • 无连接(每次连接只处理一个请求)

  • 无状态(对事务处理没有记忆能力, 如果后续处理需要前面的信息则必须重传, 这样可能会导致每次传输的数据量增大)


HTTP请求结构

请求方法: 即post, get, put, delete …

请求头: 由若干个报头组成, 每个报头的结构都是 名字 + 冒号 + 空格 + 值 (名字是大小写无关的)

这些报头用来设置http请求的一些参数

请求正文: 请求要上传的数据, 请求正文与请求头之间有空行(必须, 告诉服务端结束该信息的发送)

这是通过抓包得来的http请求的具体信息, 我把空行标记了出来


HTTP响应结构

主要有三个部分: 状态行, 响应头部, 响应正文

状态行: 协议版本 + 状态码 (200表示正常, 404 没找到页面, 5xx 服务器错误)

响应头和响应正文间也要有空行

就不再抓包了, 大致是一样的.


请求响应的步骤

  • 客户端连接到web服务器
  • 发送http请求
  • 服务器接收请求并返回http响应
  • 释放TCP连接 (如果为keep alive 则会继续保持一段时间, 该时间内可继续接受请求)
  • 客户端浏览器解析HTML内容


在浏览器地址栏输入url后, 按下回车键之后经历的流程

  1. DNS解析, 查找IP地址:

​ 浏览器缓存 -> 系统缓存(hosts文件) -> 路由器缓存 -> ips服务器缓存 -> 根域名服务器缓存 -> 顶级域名服务器缓存

​ 从左到右(由近到远)一级一级找, 如果在中间某一个位置解析到了, 则直接返回不再查询后面的缓存

  1. 建立TCP连接, 三次握手

  2. 发送http请求

  3. 服务器处理请求并返回HTTP报文
  4. 浏览器解析渲染页面
  5. 连接结束, 断开连接, 4次挥手

(5)(6) 可以看做是同时发生的


HTTP状态码

状态码由三位数字组词, 第一个数字代表响应的类别

1xx : 指示信息, 表示请求已接收, 继续处理

2xx : 成功, 表示请求已被成功接收

3xx : 重定向, 要完成请求必须进行更进一步的操作

4xx : 客户端错误, 请求有语法错误或请求无法实现

5xx : 服务端错误, 服务器未能实现合法的请求

常用状态码:

200 OK : 正常返回信息

400 Bad Request : 客户端请求有语法错误, 不能被服务器理解

401 Unauthorized : 请求未经授权, 这个状态码必须和WWW-Authenticate报头域一起使用

403 Forbidden : 服务器收到请求, 但拒绝提供服务

404 Not Found : 请求资源不存在(输入的是错误的url)

500 Internal Server Error : 服务器发生不可预期的错误

503 Server Unavailable : 服务器当前不能处理客户端的请求, 一段时间后可能恢复正常


GET请求和POST请求的区别

三个层面:

  1. http报文层面 : GET将请求信息放在URL, POST放在报文体中. 浏览器会对url的长度做出限制, 而POST由于把信息放在报文体中, 所以没有长度限制

  2. 数据库层面 : GET 符合幂等性(对数据库的一次操作和多次操作获得的结果是一致的)和安全性(对数据库中的操作没有改变数据库中的数据), POST不符合

  3. 其他层面 : GET可以被缓存, 被存储, 而POST不行(书签)


Cookie 和 Session的区别

Cookie简介

  • 是由服务器发送给客户端的特殊信息, 以文本的形式存放在客户端.
  • 客户端再次请求的时候, 会把Cookie回发
  • 服务器接收到后, 会解析Cookie生成与客户端相对应的内容

Cookie的设置以及发送过程


Session简介

  • 服务器端的机制, 在服务器上保存信息
  • 解析客户端请求并操作session id, 按需保存状态信息

Session的实现方式

使用Cookie实现, Cookie中保存着JSESSIONID, 请求的时候发给服务端

使用URL回写来实现, 客户端在发给浏览器的所有url中, 都携带JSESSIONID 这个参数


区别

  • Cookie数据存放在客户端的浏览器上, Session数据存放在服务器上
  • Session比Cookie更安全(客户端可以分析存在本地的Cookie, 进行cookie欺骗, 而session不能)
  • 若考虑减轻服务器负担, 应当使用Cookie
