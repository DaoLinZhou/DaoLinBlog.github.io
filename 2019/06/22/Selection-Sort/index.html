选择排序 Selection Sort

选择排序 (Selection Sort) O(n^2)

总结:

代码复杂程度: 简单

时间复杂度: O(n^2)

空间复杂度: O(1)

稳定性: 不稳定

特点: 交换次数最少的排序

思想: 遍历n变数组, 每一次找当前乱序部分最小的元素, 把它和乱序部分第一个元素互换


举个栗子

如果数组的元素为 {3, 8, 2, 5, 1, 4, 7}

则首先遍历一遍数组, 寻找最小元素, 最小元素为 1, 把 1 和 3 交换位置

第一次循环: {1, 8, 2, 5, 3, 4, 7} 排好序的部分为 {1} 乱序部分为 {8, 2, 5, 3, 4, 7}

第二次循环: {1, 2, 8, 5, 3, 4, 7} 互换2, 8 的位置, 排好序的部分为 {1, 2} 乱序部分为 {8, 5, 3, 4, 7}

第三次循环: {1, 2, 3, 5, 8, 4, 7} 互换3, 8 的位置, 排好序的部分为 {1, 2, 3} 乱序部分为 {5, 8, 4, 7}

第四次循环: {1, 2, 3, 4, 8, 5, 7} 互换4, 5 的位置, 排好序的部分为 {1, 2, 3, 4} 乱序部分为 {8, 5, 7}

第五次循环: {1, 2, 3, 4, 5, 8, 7} 互换5, 8 的位置, 排好序的部分为 {1, 2, 3, 4, 5} 乱序部分为 {8, 7}

第六次循环: {1, 2, 3, 4, 5, 7, 8} 互换5, 8 的位置, 排好序的部分为 {1, 2, 3, 4, 5, 7} 乱序部分为 {8}

第七次循环: {1, 2, 3, 4, 5, 7, 8} 互换8, 8 的位置, 排好序的部分为 {1, 2, 3, 4, 5, 7, 8} 乱序部分为 {}

复杂度分析:

时间复杂度:

一共有两层循环, 内层循环 j 的遍历数量受到 外层循环 i 的影响

i = 0 时, j 要从 0 遍历到 n-1, 即: n

i = 1 时, j 要从 1 遍历到 n-1, 即: n-1

i = 2 时, j 要从 2 遍历到 n-1, 即: n-2

… …

i = n-1 时, j 要从 n-1 遍历到 n-1, 即: 1

所以一共要遍历: n + (n-1) + (n-2) + … + 1 次

根据等差数列求和公式: (n+1)*n/2 = (n^2 + n)/2

而对于时间复杂度我们只关注指数最大的那一项, 而忽略常数和其他.

所以是时间复杂度为 O(n^2)


空间复杂度:

通过代码可以看到, 我们除了i, j 以外, 只声明了一个 minIndex 变量

所以空间复杂度为 O(1)

是常数级别
