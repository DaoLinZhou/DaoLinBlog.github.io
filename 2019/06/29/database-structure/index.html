数据库架构

从一个面试题说起

如何设计一个关系型数据库?


开始设计:

数据库最重要的就是存储模块(文件系统), 将数据持久化存入磁盘中

但仅仅是存储是不够的, 还要有一些响应的模块来进行管理


细分模块

  1. 存储管理: 对数据的格式, 和文件的分隔进行统一管理. 即: 把物理数据通过逻辑的形式表述出来

  2. 缓存机制: 把取出来的数据块放在缓存里, 下次需要直接从内存返回, 而不用IO

  3. SQL解析: 为了让外部可以通过SQL进行数据库操作, 需要SQL解析模块, 把sql语句解析并执行.(为了提升效率, 可以把sql 缓存到缓存里, 编译好的SQL方便下一次直接进行解析, 注意缓存不易过大, 算法里要有淘汰机制, 且sql要记录下来, 用于做主从同步或是灾难恢复)

  4. 日志管理: 对操作进行记录

  5. 权限划分: 高权限用户可以看到所有数据, 低权限用户只能看到自己应该看到的数据(多用户管理)

  6. 容灾机制: 当数据库挂了, 应该如何恢复, 恢复到什么程度

  7. 索引管理 : 优化查询效率

  8. 锁管理 : 并发操作支持

索引管理 和 锁管理 是重中之重


索引模块

常见问题:

  • 为什么要使用索引
  • 什么样的信息能成为索引
  • 索引的数据结构
  • 密集索引和稀疏索引的区别


为什么要使用索引?

最简单的方式进行数据查询 : 全表扫描

把整张表的数据,全部 or 分批次加载到内存当中

之后逐块进行轮询, 知道找到想要的数据, 这种方式在小数据量时性能还可以接受, 但大数据量的情况下非常缓慢.

因此为了避免全表扫描, 所以应该使用一种更高效的机制(索引), 就像字典中偏旁部首一样, 而索引可以帮助数据库快速查询数据


什么样的信息能成为索引

主键, 唯一键以及普通键等…


索引的数据结构

生成索引, 建立二叉查找数进行二分查找, 以及变种平衡二叉树/ 红黑树

生成索引, 建立B-Tree结构进行查找

生成索引, 建立B+-Tree结构进行查找 (mysql实现方式)

生成索引, 建立Hash结构进行查找
